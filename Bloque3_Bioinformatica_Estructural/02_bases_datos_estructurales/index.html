    Bases de datos estructurales

    Introducción a las bases de datos estructurales

    Las bases de datos estructurales recopilan, organizan y ponen a disposición del público estructuras tridimensionales de macromoléculas como proteínas, ADN, ARN y complejos macromoleculares. Constituyen una herramienta esencial para el estudio de la bioinformática estructural, ya que permiten analizar el plegamiento molecular, la función biológica y las interacciones entre biomoléculas.

    A diferencia de las bases de datos de secuencias, que almacenan información lineal, estas bases recogen información espacial: coordenadas atómicas, mapas de densidad y metadatos experimentales.

    Tipos de bases de datos estructurales

    Bases de datos primarias

    Las bases de datos primarias almacenan estructuras determinadas experimentalmente y depositadas por investigadores. No realizan anotaciones derivadas ni clasificaciones complejas. Su función principal es conservar el dato experimental.

    Ejemplos: - PDB (Protein Data Bank)
    - EMDB (Electron Microscopy Data Bank)
    - BMRB (BioMagResBank)

    Bases de datos secundarias

    Las bases de datos secundarias realizan análisis, clasificación o anotación de las estructuras presentes en bases primarias. Producen información derivada como familias estructurales, pliegues, topologías o diagramas funcionales.

    Ejemplos: - SCOP
    - CATH
    - PDBsum

    Protein Data Bank (PDB)

    Historia y organización

    El PDB es la base de datos estructural más importante. Fundada en 1971, hoy es gestionada por un consorcio internacional (RCSB, PDBe, PDBj y BMRB). Su misión es recopilar, validar y difundir estructuras tridimensionales de macromoléculas.

    Contenido del PDB

    Incluye: - Proteínas
    - ADN y ARN
    - Complejos macromoleculares
    - Fármacos, ligandos y compuestos asociados
    - Estructuras complejas como ribosomas o proteosomas

    Formatos del PDB

    • PDB: formato clásico en texto plano
    • mmCIF: formato moderno con metadatos ampliados
    • MRC: mapas volumétricos para Cryo-EM

    Proceso de depósito

    El proceso incluye validación automática, revisión curatorial y asignación de un identificador único PDB de 4 caracteres.

    EMDB: Electron Microscopy Data Bank

    La EMDB recopila mapas tridimensionales obtenidos mediante criomicroscopía electrónica (Cryo-EM). Mientras que PDB almacena modelos atómicos, EMDB contiene los mapas de densidad que sirven como base para construir dichos modelos.

    Incluye: - Mapas de densidad
    - Metadatos experimentales
    - Información sobre resolución, FSC y condiciones experimentales

    Cuando existe un modelo asociado, suele estar vinculado al PDB.

    BMRB: Banco de RMN (BioMagResBank)

    El BMRB almacena datos experimentales procedentes de resonancia magnética nuclear (RMN). Entre ellos: - Espectros
    - Constantes de acoplamiento
    - Restricciones para el cálculo estructural

    Complementa al PDB, especialmente para proteínas pequeñas o en solución.

    NDB: Nucleic Acid Database

    La NDB es una base especializada en estructuras de ADN y ARN. Incluye: - Conformaciones de ADN en distintas formas (A, B, Z) - Motivos estructurales de ARN - Datos cristalográficos específicos - Clasificación y análisis de motivos estructurales en ácidos nucleicos

    Bases de datos secundarias: clasificación y anotación

    SCOP

    SCOP clasifica proteínas según relaciones evolutivas y estructurales en una jerarquía: - Familia
    - Superfamilia
    - Pliegue
    - Clase

    CATH

    CATH combina algoritmos automáticos con revisión manual. Su estructura jerárquica es: - Class
    - Architecture
    - Topology
    - Homologous superfamily

    PDBsum

    PDBsum ofrece resúmenes visuales y funcionales: - Diagramas 2D
    - Interacciones moleculares
    - Estadísticas estructurales
    - Accesibilidad superficial y dominios

    Contenido y organización de las entradas

    Una entrada estructural incluye: - Información experimental
    - Archivos principales (PDB, mmCIF, mapas)
    - Indicadores de calidad (R-factor, R-free, resolución)
    - Ligandos, metales y moléculas asociadas
    - Ensamblajes biológicos frente a cristales

    Herramientas de búsqueda y navegación

    Las bases de datos permiten búsquedas por: - Organismo - Técnica experimental
    - Tipo de molécula
    - Ligandos
    - Identificador PDB
    - Palabras clave

    También incluyen métodos de búsqueda estructural: - Comparación 3D - Alineamientos espaciales - Detección de similitudes en dominios y plegamientos

    Importancia de las bases de datos estructurales

    Son fundamentales para:
    - Validar modelos de predicción estructural
    - Diseño racional de fármacos
    - Análisis funcional de proteínas y ácidos nucleicos
    - Estudiar efectos de mutaciones en la estructura
    - Integrar datos con otras bases ómicas como genómica y proteómica

    Resumen del tema

    Las bases de datos estructurales permiten almacenar, analizar y clasificar estructuras tridimensionales. Incluyen bases primarias como PDB, EMDB y BMRB, y bases secundarias como SCOP, CATH y PDBsum. Su uso es esencial en investigación, docencia y biotecnología.
